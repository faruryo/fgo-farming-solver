## 2026-07-21 追記 — スクリーンショットOCRによる所持数取り込みを実装（下記2026-06-16の判断を更新）

2026-06-16時点の見送り理由のうち、「ページング地獄」はユーザーストーリーの前提（複数枚まとめて投稿・統合できるレビューUI）で解消可能と判断し、`material-possession-screenshot-import` change として実装した。

- **方式**: ブラウザ内完結のクライアント側OCR（`ppu-paddle-ocr`、PP-OCRv6統合多言語モデル、onnxruntime-web）。マルチモーダルLLM画像直読みは採用せず（サーバーコストが発生するため、再検討条件は変更なし）。
- **精度検証で判明した重要な事実**:
  - 当初調査していたPaddleOCR系モデル（`ch_PP-OCRv4`、中国語版）はカタカナをほとんど認識できず、マスターデータの約69%を占めるカタカナ含みアイテム名を識別できないことが判明。PP-OCRv6統合多言語モデルに切り替えて解決。
  - カード検出ロジックの色閾値バグにより、実アプリへの統合E2Eテストをして初めて「全アイテムの所持数が読み取れない」重大な不具合を発見（Pythonでのプロトタイプ検証・個別ライブラリの単体検証では検出できなかった）。**個別要素の精度検証だけでなく、実データを使った実アプリへの統合E2E検証が本質的に必要**という教訓が得られた。
- **結果**: 実際のマイルーム所持アイテム一覧スクリーンショット2枚（計18アイテム、末尾2件は画像端で見切れ）で検証し、見切れていない16アイテム全てで品目名・所持数とも完全一致。確定操作で `posession`（`localStorage`）が更新され、クエスト効率ランキングへの反映も実機確認済み。
- **保守コストの見通し**: ゲームUI変更への耐性は、品目名OCR結果を既知アイテム名辞書へファジーマッチする方式のため比較的高いと見込むが、実運用でのモデル陳腐化・精度劣化は未検証（継続監視が必要）。

詳細は `openspec/changes/material-possession-screenshot-import/design.md` を参照。

### 再検討の条件（更新）
- 実運用でファジーマッチ・カード検出の精度が実用に耐えないと分かった場合、方式を見直す。
- iPhone Safari実機での動作・処理時間が許容範囲を超える場合、Decision 5（COEP不採用）を再検討する。

## 2026-06-16 — 所持アイテムデータの自動取り込みを当面見送る（上記2026-07-21で実装・更新）

**ステータス**: 見送り（Deferred）

### 背景
競合（特に Chaldea）は、所持素材を手入力せずに取り込む機能を持つ。これが乗り換え障壁になりうるため、本プロダクトでも実現可能か検討した。検討した方式は次の2つ。

1. **スクリーンショット OCR** — 所持アイテム一覧のスクショを解析（アイコンの perceptual hash 照合 + 数字認識）。
2. **HTTPS スニッフィング** — ゲームログイン時の通信を傍受し、サーバーが返す JSON を丸ごと取得。

### 決定
**どちらも当面実装しない。** データ自動取り込み機能はスコープ外とする。

### 理由
- **スクショ OCR はページング地獄**: FGO の素材一覧は1画面に少数しか表示されず、全素材を撮るには何枚もスクショして連続アップロードが必要。手入力より面倒になりかねず本末転倒。先行する Chaldea もスクショ方式を保守終了している（参考: https://docs.chaldea.center/guide/import_data ）。
- **HTTPS スニッフィングは JP iPhone 非対応**: Chaldea のドキュメント（ https://docs.chaldea.center/guide/import_https/ ）の通り JP/iPhone はサポート外で、開発者本人の環境で使えない。自分が使えない機能を継続保守するのは現実的でない。
- **コア体験のレバーではない**: 本プロダクトは既にクラウド同期で所持数を永続化している。所持数の入力は「毎回」ではなく「変わった分だけ更新」であり、手入力の苦痛は実質ワンタイムに償却される。自動取り込みは乗り換え障壁を下げる効果はあっても、コア体験を変える機能ではない。
- **保守コストが高い**: 認識系はゲーム UI 更新・新素材追加・新端末のアスペクト比で壊れる。継続メンテの負担が大きい。

### 再検討の条件
次のいずれかが成立したら再検討する。
- 他ツール（Chaldea / FGO Manager 等）のエクスポートファイル取り込みのような、**低コストで「手入力したくない」を満たす手段**を入れたくなったとき（OCR より圧倒的に安い）。
- マルチモーダル LLM による画像直読みの**精度・コスト・レイテンシが無料運用と両立**するレベルに改善したとき。
- **JP iPhone でも使える**スニッフィング/連携手段が確立したとき。
