## MODIFIED Requirements

### Requirement: アイテム入手による残りの減少（目標固定再ソルブ）
システムは、比較スナップショット以降にユーザーがアイテムを入手したことで「残りに必要なAP・周回数」がどれだけ減ったかを、目標を現在に固定した再ソルブで算出し、**参考の副指標**として表示しなければならない (SHALL)。目標を両辺とも現在値に固定するため、目標を増やしても本指標は増加してはならない (SHALL NOT)。本指標は進捗 tier の判定根拠としては用いず、値が 0 以下のとき（素材を育成等で消費した日など）は表示しない (SHALL NOT)。

#### Scenario: アイテム入手による減少量の算出
- **WHEN** 現在の目標（`material/result`）と、現在の所持（`posession`）および比較スナップショットの過去所持が取得できるとき
- **THEN** `need = max(0, 現在目標 − 所持)` を所持別に作り、現在所持版と過去所持版をそれぞれソルバーで解く。
- **THEN** 減少AP `= solve(現在目標 − 過去所持).total_ap − solve(現在目標 − 現在所持).total_ap`、減少周回 `= 同 total_lap`、減少費用 `= round(減少AP / 144 / 168 * 10000)` を算出する。

#### Scenario: 目標増加が減少量に影響しないこと
- **WHEN** 比較スナップショット以降に育成目標が増えた（`material/result` の総数が増加した）とき
- **THEN** 目標は両辺とも現在値で固定されているため、目標増加そのものは減少AP・減少周回を増減させない。

#### Scenario: 減少量が 0 以下のときの非表示
- **WHEN** 比較スナップショット以降に素材を育成等で消費し、減少AP が 0 以下になったとき
- **THEN** 「残りAP −○」等の減少指標は表示せず、進捗 tier も none に引きずらない（tier は素材スループットで判定する）。

#### Scenario: 過去所持が欠損している場合のフォールバック
- **WHEN** 比較スナップショットに `posession` が記録されていないとき
- **THEN** 減少AP・減少周回・減少費用は算出せず非表示とし、他指標のみ表示する。

## ADDED Requirements

### Requirement: 素材スループットによる進捗判定
システムは、比較スナップショット以降の**素材の活動量**を、所持数の増減から算出し、進捗 tier・見出し・マシュのメッセージの主たる判定根拠として用いなければならない (SHALL)。活動量は「獲得素材（所持が増えた分の合計個数）」と「育成投入（所持が減った分の合計個数）」からなり、両者の合計（スループット）で評価する。QP（atlasId `1`）は所持規模が他素材を桁違いに上回るため、本指標から除外しなければならない (SHALL)。育成目標の増減は本指標に影響させてはならない (SHALL NOT)。

#### Scenario: 素材を育成に消費した日の評価
- **WHEN** 比較スナップショット以降に、ユーザーが素材を育成（霊基再臨・スキル強化等）に消費し、所持数が純減したとき
- **THEN** 消費した個数の合計を「育成投入」として進捗に加算し、tier を none 以外に判定しうる。
- **THEN** マシュは「お疲れさま（休んでください）」系ではなく、活動を労うメッセージを表示する。

#### Scenario: 素材を周回で獲得した日の評価
- **WHEN** 比較スナップショット以降に、ユーザーが周回で素材を獲得し、所持数が純増したとき
- **THEN** 増えた個数の合計を「獲得素材」として進捗に加算する。

#### Scenario: QP の除外
- **WHEN** 所持差分に QP（atlasId `1`）の増減が含まれるとき
- **THEN** QP の増減はスループットに含めない。

#### Scenario: 活動が無い日のフォールバック
- **WHEN** 素材スループットが 0 で、育成総量・新規入手・減少AP もいずれも無いとき
- **THEN** 進捗なし（`zero_progress`）として、登録を労うフォールバックメッセージを表示する。
