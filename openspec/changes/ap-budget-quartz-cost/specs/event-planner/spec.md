## ADDED Requirements

### Requirement: AP回復アイテムと聖晶石による消費AP内訳
システムは、計画の必要AP を、ユーザーの所持AP回復アイテム（りんご各種）と聖晶石に割り当てた内訳として提示しなければならない (SHALL)。AP の回復量はマスターレベル由来の最大AP に基づき、固定換算（旧 `金果実=40AP / 銀林檎=20AP`）を用いてはならない (SHALL NOT)。

#### Scenario: りんご所持数を反映した内訳
- **WHEN** ユーザーがマスターレベルと所持りんご各種の枚数を入力したとき
- **THEN** 必要AP に対して所持りんごを消費した内訳（種類ごとの消費数）と、賄いきれず残るAP を表示する。

#### Scenario: 不足分の聖晶石換算
- **WHEN** 所持りんごで必要AP を賄いきれないとき
- **THEN** 残りのAP を聖晶石（1個＝最大APの全回復）に換算した必要枚数を表示する。

#### Scenario: 所持りんご枚数の端末ローカル永続
- **WHEN** ユーザーが所持りんご枚数を入力したとき
- **THEN** 値は localStorage に永続するが `use-cloud-sync` の同期対象には含めない（端末ローカルのみ）。

### Requirement: 聖晶石必要枚数の課金額換算
システムは、必要な聖晶石枚数を、App Store の最大金額パックの単価で課金額（円）に換算して提示しなければならない (SHALL)。階段価格の最適化（分割購入・福袋・セール）は行わない。

#### Scenario: 最大パック単価での金額化
- **WHEN** 聖晶石の必要枚数が算出されたとき
- **THEN** App Store 最大パックの単価（現行 ¥10,000 / 168個）を一定単価として掛けた目安金額を表示する。

#### Scenario: 換算式のインフォメーション表示
- **WHEN** ユーザーが課金額表示の根拠を確認しようとしたとき
- **THEN** 「不足AP→聖晶石全回復換算→App Store最大パック単価で金額化、実最安は分割/セールで変動」という換算式の説明をインフォメーション（ツールチップ等）で確認できる。

### Requirement: 終了済みイベントのシミュレート
システムは、開催中イベントに加えて、終了済みイベントについても計画ページでのシミュレートを許可しなければならない (SHALL)。

#### Scenario: 終了済みイベントの選択
- **WHEN** ユーザーが一覧から終了済みのイベントを選んだとき
- **THEN** そのイベントの計画ページが開き、開催中と同一の入力・算出が行える。

#### Scenario: 会期外の明示
- **WHEN** 終了済みイベントの計画ページを開いたとき
- **THEN** 会期が終了している旨を明示する。
