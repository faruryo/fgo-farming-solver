# Delta: progress-visualizer (progress-5tier-lap-value)

## MODIFIED Requirements

### Requirement: 進捗 tier の判定
システムは、進捗の達成度を `legendary` / `large` / `medium` / `small` / `none` の 5 段階（tier）で判定しなければならない (SHALL)。主指標は「バッファ込み実効不足の解消ぶんの推定周回数（前進周回）」を経過日数でならした値（周/日）とする。

前進周回は次で算出しなければならない (SHALL): 素材ごとに 単価（1個あたり推定周回数）= `1 ÷ ユーザー選択クエスト内の最高ドロップ率`（選択クエスト内にドロップが無い素材は全クエスト中の最高率にフォールバック）、実効目標 = `material/result + (stockEnabled ? ストックバッファ : 0)`（quest-efficiency の `effectiveRequired` と同一定義）、現在所持は過去所持で下限クランプ（消費中立）した上で、`(実効不足の過去値 − 現在値) × 単価` を合算する。入手経路（周回・イベント交換・ストーリー報酬）は問わない。

しきい値（周/日）: `≥60 → legendary`、`≥15 → large`、`≥5 → medium`、`>0 → small`、`0 → none`（補完判定へ）。前進周回が 0 の期間は、労力周回（獲得素材の周回換算）に同じしきい値を適用して tier を補完しなければならない (SHALL)。ただし補完による tier は `large` を上限とし、`legendary` は前進周回によってのみ到達できる (SHALL)。

#### Scenario: 周回換算による判定
- **WHEN** 前進周回の周/日が算出できたとき
- **THEN** `≥60 → legendary`、`≥15 → large`、`≥5 → medium`、`>0 → small` と判定する。

#### Scenario: バッファ込み実効不足への前進
- **WHEN** 育成目標は充足済みだが、所持がストックバッファ込みの実効目標に満たない素材を獲得したとき
- **THEN** その獲得分も前進周回に計上する（stockEnabled=OFF のユーザーはバッファ0として従来の育成目標のみで判定する）。

#### Scenario: 育成消費が tier を下げないこと
- **WHEN** 同一期間内に素材の獲得と育成による消費の両方が発生したとき
- **THEN** 消費は前進周回を減少させず、獲得のみが tier 判定に寄与する。

#### Scenario: 選択クエストにドロップが無い素材の単価
- **WHEN** 不足解消した素材のドロップ元がユーザーの選択クエストに含まれないとき
- **THEN** 全クエスト中の最高ドロップ率で単価を算出し、0 扱いにしない。

#### Scenario: 前進ゼロ時のスループット補完
- **WHEN** 前進周回が 0 で、獲得素材（余剰込み）の労力周回が正のとき
- **THEN** 労力周回に同じしきい値を適用して tier を補完する。ただし `legendary` には到達させない（上限 `large`）。

#### Scenario: 進捗ゼロの判定
- **WHEN** 前進周回・労力周回・育成総量・新規入手のいずれも無いとき
- **THEN** tier は `none`、フォールバックは `zero_progress` と判定する。

### Requirement: 目標への前進（アイテム入手による残りの減少）
システムは、比較スナップショット以降の「バッファ込み実効不足の解消ぶん」を推定周回数（前進周回）に換算し、「目標への前進 +N周ぶん」として**周回数を主役**に**プラス表記**で表示しなければならない (SHALL)。内訳として AP相当（素材ごとの最安AP単価 = `選択クエスト内の最小 AP÷ドロップ率` による独立換算の合計）と費用（`round(AP相当 / 144 / 168 * 10000)` 円）を小さく併記する。

算出は**消費中立**でなければならない (SHALL): アイテムごとに現在所持を過去所持で下限クランプしてから不足の減少を数え、育成等による所持の純減は本指標を減少させない。これにより前進周回・AP相当は常に 0 以上となる。素材ごとの独立換算（単価×解消数の合算）であり、ソルバーの再実行を必要としない (SHALL NOT)。本指標は tier 判定の主指標と同一である。

#### Scenario: 前進の算出
- **WHEN** 現在の目標（`material/result`）・ストック設定（`efficiency/stockBuffer`・`efficiency/stockEnabled`）・現在所持・比較スナップショットの過去所持・drops（ドロップ率）が取得できるとき
- **THEN** 素材ごとに `実効目標 = material/result + (stockEnabled ? buffer : 0)`、`adjustedNow = max(現在所持, 過去所持)` を作り、`(max(0,実効目標−過去所持) − max(0,実効目標−adjustedNow)) × 単価` を合算して前進周回と AP相当を算出する。

#### Scenario: 周回主役での表示と内訳
- **WHEN** 前進周回 > 0 のとき
- **THEN** 「目標への前進 +N周ぶん」を主役として表示し、その下に AP相当と費用を小さく内訳表示する。

#### Scenario: 育成消費が前進を目減りさせないこと
- **WHEN** 比較スナップショット以降に素材を獲得しつつ、育成で一部素材の所持が純減したとき
- **THEN** 純減した素材は過去所持と同値として扱われ、獲得（純増）した素材の分だけが前進に計上される。前進が負になってはならない。

#### Scenario: 過去所持が欠損している場合のフォールバック
- **WHEN** 比較スナップショットに `posession` が記録されていないとき
- **THEN** 前進（周回/AP/費用）は算出せず非表示とし、tier は労力周回等で判定する。

### Requirement: 素材スループットの算出
システムは、比較スナップショット以降の所持数の増減から素材スループットを算出しなければならない (SHALL)。所持が純増した素材の合計個数を「獲得素材（`itemsFarmed`）」、純減した合計個数を「育成投入（`itemsConsumed`）」とし、QP（atlasId `1`）は除外する。

さらに、獲得素材を素材ごとの単価（1個あたり推定周回数）で換算した合計を「労力周回（推定活動量）」として算出しなければならない (SHALL)。労力周回は余剰在庫行きの獲得も含み、育成投入は含まない（育成の認知は育成総量・サーヴァント成長表示が担う）。労力周回は (a) 「この期間の活動」での「推定活動量 約N周相当」表示、(b) 前進ゼロ時の tier 補完、(c) マシュメッセージの2軸修飾に用いる。

#### Scenario: QP の除外
- **WHEN** 所持差分に QP（atlasId `1`）の増減が含まれるとき
- **THEN** QP の増減はスループット・労力周回に含めない。

#### Scenario: 労力周回の表示
- **WHEN** 労力周回が正のとき
- **THEN** 「この期間の活動」に「推定活動量 約N周相当」を表示する。

#### Scenario: 育成投入は労力に含めない
- **WHEN** 所持が純減した素材（育成投入）があるとき
- **THEN** その換算周回は労力周回に加算しない。

### Requirement: マシュによる進捗フィードバック
進捗報告画面において、マシュ・キリエライトの画像と進捗種別に応じたメッセージを表示しなければならない (SHALL)。メッセージは方向性（tier）と労力（労力周回）の2軸で選択・修飾しなければならない (SHALL)。

#### Scenario: tier 別マシュメッセージ
- **WHEN** 進捗 tier が `legendary` / `large` / `medium` / `small` / `none` のいずれかに判定されたとき
- **THEN** その tier に対応する事前定義済みマシュメッセージ群からランダムにメッセージが選択され、マシュの画像とともに表示される。`legendary` はボックス/レイド全力月に相応しい最大級の称賛トーンとする。

#### Scenario: 労力による修飾
- **WHEN** tier が `medium` 以下で、労力周回が `large` 相当以上のとき
- **THEN** 活動量・備蓄を労うトーンのメッセージを優先する（「たくさん動いた月」を認知する）。

#### Scenario: 「お疲れ様」モード
- **WHEN** 過去スナップショットがない、または全指標で改善ゼロのとき
- **THEN** マシュの「登録お疲れ様」系メッセージが表示される。

#### Scenario: 新規入手・育成は tier に依らず労う
- **WHEN** 新規入手サーヴァントが居る、または育成総量が正のとき
- **THEN** tier が `none` であっても「休んでください」系ではなく、新規入手の歓迎／育成を労うメッセージを優先して表示する。

### Requirement: 達成感の視覚的演出
進捗 tier（5段階）に応じて、パネル内の色・サイズ・装飾の強度を変化させなければならない (SHALL)。`legendary` は `large` を上回る特別な演出とする。アニメーションは v1 では実装しない。

#### Scenario: legendary のビジュアル
- **WHEN** tier が `legendary` のとき
- **THEN** `large` よりさらに強い特別な配色・装飾で最大級の達成感を演出する。

#### Scenario: tier 別ビジュアル
- **WHEN** tier が `large` のとき
- **THEN** 派手な配色と大きい数値表示で達成感を強調する。

#### Scenario: 控えめな表示
- **WHEN** tier が `small` または `none` のとき
- **THEN** 落ち着いた配色と通常サイズで控えめに表示する。
