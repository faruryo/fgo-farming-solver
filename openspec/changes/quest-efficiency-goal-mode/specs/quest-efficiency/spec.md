## MODIFIED Requirements

### Requirement: 所持数と目標による個人最適化(2段階重み)

システムは各素材の重みを所持数(`posession`)・目標(`items`)・レア別余剰しきい値から決定 SHALL。対象モードは「全部」「不足のみ」「不足+余剰ストック」の3値とする。

「不足のみ」モードでは次のとおり2段階で決定する:
- `owned < goal`(不足) → 重み **1**(主優先)
- 上記以外で余剰 `owned - goal` がレア別余剰しきい値以下 → 重み **0.3**(次点)
- 余剰がしきい値を超える → 重み **0**

「不足+余剰ストック」モードでは、レア別余剰しきい値を**ストック個数として目標に上乗せ**し、実効目標 `effGoal = goal + stock[rarity]` で1段階判定する:
- `owned < effGoal` → 重み **1**
- `owned >= effGoal` → 重み **0**
このモードでは次点(0.3)バンドは使わない。`stock` は「レア別余剰しきい値」の値を流用する。

レアリティは item の `category` から判定 SHALL。レアリティ不明の素材はストック上乗せの対象外(`stock=0` 相当)とする。QP・絆はドロップデータに存在しないため効率ポイントに寄与しない(「QP 不要」は自動成立)。「全部」モードでは全対象素材を重み1とする SHALL。

#### Scenario: 不足は主優先
- **WHEN** ある素材の所持数が目標未満で、不足のみモードが有効
- **THEN** その素材は重み1で効率ポイントに寄与する

#### Scenario: 充足済みでも余剰が少なければ次点
- **WHEN** ある素材の所持数が目標以上だが、余剰がそのレアリティのしきい値以下で、不足のみモードが有効
- **THEN** その素材は重み0.3(次点)で効率ポイントに寄与する

#### Scenario: 余剰が多ければ除外
- **WHEN** ある素材の余剰がそのレアリティのしきい値を超え、不足のみモードが有効
- **THEN** その素材の重みは0となり効率ポイントに寄与しない

#### Scenario: 余剰ストックを目標に上乗せ
- **WHEN** 「不足+余剰ストック」モードが有効で、ある素材の所持数が `育成必要数 + レア別ストック個数` 未満である
- **THEN** その素材は重み1(主優先)で効率ポイントに寄与する

#### Scenario: ストック込み目標に到達したら除外
- **WHEN** 「不足+余剰ストック」モードが有効で、ある素材の所持数が `育成必要数 + レア別ストック個数` 以上である
- **THEN** その素材の重みは0となり効率ポイントに寄与しない(次点0.3も付かない)

#### Scenario: QP 不要の表現
- **WHEN** 効率ポイントを算出する
- **THEN** QP・絆はドロップデータに存在しないため寄与せず、QP 偏重クエストの効率ポイントは相対的に低くなる

#### Scenario: 全部モード
- **WHEN** 全部モードが有効
- **THEN** 所持数・目標・余剰しきい値に関わらず、ドロップする全素材が効率ポイントに寄与する

### Requirement: レア別余剰しきい値

システムは余剰しきい値を金/銀/銅(`background` の `gold`/`silver`/`bronze`)ごとに保持 SHALL。妥当なデフォルト値を持ち、ユーザーが数値で上書きでき、上書き値はクラウド同期される SHALL。この値は対象モードによって意味が切り替わる: 「不足のみ」では次点(0.3)で拾う余剰の上限、「不足+余剰ストック」では目標に上乗せするストック個数として解釈される。値とストレージ(`efficiency/surplusThreshold`)は両モードで共有 SHALL。

#### Scenario: デフォルトで動作
- **WHEN** ユーザーがしきい値を未設定のまま一覧を開く
- **THEN** レア別のデフォルトしきい値で次点判定(不足のみ)またはストック上乗せ(不足+余剰ストック)が行われる

#### Scenario: ユーザー調整
- **WHEN** ユーザーが金/銀/銅の余剰しきい値を変更する
- **THEN** 次点に拾われる素材の範囲(不足のみ)または目標へ上乗せされるストック個数(不足+余剰ストック)が変わり、効率ポイントが再計算される

#### Scenario: 両モードで同一値を共有
- **WHEN** ユーザーが余剰しきい値を変更し、対象モードを切り替える
- **THEN** どちらのモードでも同じ `efficiency/surplusThreshold` の値が用いられる(モードごとの別設定は持たない)

## ADDED Requirements

### Requirement: 対象モードの3値選択(上級者向け目標モード)

システムは `/quests` のフィルターポップオーバーで対象モードを「全部 / 不足のみ / 不足+余剰ストック」の3値から選択できる SHALL。既定は「不足のみ」とし、「不足+余剰ストック」は育成達成が近い上級者向けのオプトインとして、一般ユーザーの既定挙動・主行レイアウトを変えない。選択状態は永続化(localStorage)され、旧 `quests/efficiency/shortageOnly`(boolean)からは `true→不足のみ` / `false→全部` に読み替える SHALL。所持数モーダルの余剰しきい値の説明文は、現在の対象モードに応じて「次点の上限」か「目標へ上乗せするストック個数」かを出し分ける SHALL。

#### Scenario: 3値モードの選択と再計算
- **WHEN** ユーザーが対象モードを「不足+余剰ストック」に切り替える
- **THEN** 効率ポイントとランキングが、ストックを上乗せした目標で即座に再計算される

#### Scenario: 旧設定の読み替え
- **WHEN** 旧 `quests/efficiency/shortageOnly` のみが保存された状態でページを開く
- **THEN** `true` は「不足のみ」、`false` は「全部」として対象モードが初期化される

#### Scenario: 説明文のモード連動
- **WHEN** 対象モードが「不足+余剰ストック」で所持数モーダルを開く
- **THEN** 余剰しきい値が「目標へ上乗せするストック個数」として説明される
