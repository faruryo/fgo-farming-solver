## Context

FGOの育成素材計算結果画面 (`/material/result`) では、ユーザーの不足素材が一覧表示されますが、配布素材の選択や交換券の割り当てにおける「どれを優先すべきか」を論理的に判断する機能はありません。
また、フリクエの効率には「AP効率」と「周回数効率」の2つの側面があり、ユーザーのニーズ（果実/石を節約したいか、プレイ時間を節約したいか）によって最適な割り当ては変化します。
本設計は、これらの課題を解消するための汎用的かつ軽量なクライアントサイドの素材選択アドバイザー（Material Selection Advisor）を構築するものです。

## Goals / Non-Goals

**Goals:**
- **汎用的な選択アドバイザーUIの提供**: ユーザーが素材候補を任意に追加・削除でき、獲得可能総数を指定して最適な交換配分を確認できる。
- **2つの評価軸（AP / 周回数）のサポート**: AP節約優先モードと周回数節約優先モードの切り替えをサポート。
- **論理的な貪欲法エンジン**: クライアントサイドで一瞬で完結する軽量な最適化計算ロジック。
- **マシュのアドバイスセクション**: 最適解に基づき、最もお得なアイテムや削減AP/周回数をマシュがチャット枠で解説してくれる premium な演出。
- **設定の永続化**: 選択した素材候補、総数、モードは `localStorage` に自動保存され、リロード後も復元される。

**Non-Goals:**
- **新規のドロップデータ収集**: 既存の `Drops` （`getDrops()` から得られるマスターデータ）をそのまま利用し、外部APIを新規追加しない。
- **周回ソルバー（LP Solver）の変更**: `/farming` のLP計算エンジンには一切変更を加えない。
- **ゲーム本体との自動連携**: ゲームにログインして自動交換するような自動化機能はスコープ外。

## Decisions

### 1. クライアントサイドでの貪欲法（Greedy Algorithm）による最適化
**意思決定のRationale**:
今回の最適化問題は、候補素材数 $M \le 10$ 程度、獲得可能総数 $N \le 100$ 程度の非常に小規模なものです。この規模であれば、重い線形計画法（LP Solver）を用いずとも、削減AP（または削減周回数）が大きい順にソートして不足分を割り振る「貪欲法」で完全に厳密な最適解が得られます。
計算量は $O(M \log M)$ であり、ブラウザ上でリアルタイムに入力値が変わるたびに超高速（ミリ秒未満）で再計算可能です。

**代替案**:
- `/farming` のLP Solverに組み込む：交換券という概念をソルバーの制約に組み込むことも技術的には可能ですが、ユーザーが「配布だけでどうするか」をアドバイスしてほしいという独立したニーズには、別の専用UIと軽量ロジックを設けた方が圧倒的に使いやすいため見送りました。

### 2. 素材のドロップ効率およびフリクエAP/個・周回/個の逆算方法
既存の `Drops` データには、クエストごとのドロップ率 (`drop_rate`) と AP (`ap`) が含まれています。
各アイテムについて：
- **AP効率最小化時のコスト**:
  $$Cost_{i,\text{AP}} = \min_{q} \left( \frac{ap_q}{drop\_rate_{q,i}} \right) = \frac{1}{\max_{q} (drop\_rate_{q,i} / ap_q)}$$
- **周回数最小化時のコスト**:
  $$Cost_{i,\text{周回}} = \frac{1}{\max_{q} (drop\_rate_{q,i})}$$

上記を `getDrops()` のデータから動的に抽出し、各アイテムの「フリクエで1個集めるのに必要なAPまたは周回数」を決定します。
※ドロップデータが存在しないアイテム（QP、ピース、モニュメント、その他フリクエで落ちないアイテム）については、 $Cost_i = 0$ または例外処理（アドバイザー上で「フリクエ対象外」として警告表示する等）としてフォールバックします。

### 3. UIの配置および統合箇所
**意思決定のRationale**:
このアドバイザー機能は `/material/result` （アイテム必要数）の直下、あるいは同じコンポーネント内のタブ/トグルエリアに設置します。
ユーザーが「アイテム必要数」画面で自分の不足素材を確認した直後に、シームレスに「今月の交換券や配布でどれをもらうか」を設定・検討できるため、最も摩擦の少ない導線になります。

## Risks / Trade-offs

### [Risk 1] ドロップデータが存在しないアイテム（QPや特殊素材）が選択された場合のゼロ除算・計算不能
- **Mitigation**:
  - `bestEff` が 0 または算出できないアイテムについては、AP/周回数の節約期待値を `0` （または判定不能）として扱い、UI上で「このアイテムはフリクエで恒常ドロップしないため、優先度計算から除外されます」といった注意書きを表示します。

### [Risk 2] 複数の月や異なる配布イベントの設定の競合
- **Mitigation**:
  - 汎用的な設定として `localStorage` に1組の設定状態 (`material/selection-advisor-config`) を保持しますが、ユーザーがいつでもリセット（全削除）して新しい配布イベント用に設定をクリアできる「スロットのリセット」ボタンをUIに配備します。
