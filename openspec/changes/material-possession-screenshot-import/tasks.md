## 0c. 動的グリッド検出の実画像検証（完了・使い捨てプロトタイプ）

3.1 実装直後、Playwright経由で実スクリーンショット2枚（IMG_9784/9785、2796×1290）に対して実行し、可視カードと突き合わせて検証した。

- [x] 0c.1 初回実装をそのまま実行 → 暗い色のアイコン（虚影の塵・宵哭きの鉄杭）がカード内に約9pxの局所的低密度ディップを作り、1枚のカードを2つに誤分割する不具合を発見（隙間結合幅8pxでは吸収できず、実際のカード間隙間は約34px）
- [x] 0c.2 バンド結合幅を8px→20pxに調整（誤分割の原因である9px幅ディップは吸収しつつ、実際のカード間隙間34pxは分離を維持）し、誤分割が解消することを確認
- [x] 0c.3 画像下端で見切れた最終行が、ピクセル境界に厳密に接していない（数px程度の余白がある）ため `clipped` フラグが立たない不具合を発見
- [x] 0c.4 内側バンドの標準サイズに対する比率ベースの判定を追加（標準サイズの85%未満の先頭/末尾バンドは見切れとみなす）し、修正を確認
- [x] 0c.5 修正後、実スクリーンショット2枚で検出したカード領域をそのままOCR（`ppu-paddle-ocr`）にかけるフルパイプライン検証を実施 → 品目名は shot1: 8/8、shot2: 可視8/8 とも正読し、末尾2カード（励振火薬・赦免の小鐘）は正しく `clipped=true` として検出された。**この時点では出力を品目名の正誤のみで確認しており、所持数が一切抽出されていなかったことを見落としていた**（0d参照。実際にはこの回帰は既に作り込まれていた）。

## 0d. 実アプリ統合E2E検証で発見した重大な不具合（完了）

3〜5実装後、`pnpm dev` を起動し実際のブラウザ（Chrome、`/quests` ページの `PossessionModal` から）で実スクリーンショット2枚をアップロードして検証したところ、**全アイテムの所持数候補が常に空（要確認）になる**不具合を発見した。

- [x] 0d.1 レビューUIで全18カードが「要確認」・所持数欄が空欄になることを確認。デバッグログでOCR結果を確認したところ、各カードにつき品目名の1行しか返っておらず、「Item」バッジ・「所持」ラベル・所持数がまったく含まれていないと判明
- [x] 0d.2 原因調査の過程で、実行環境の違い（実ChromeはWebGPU利用可能、それまでの検証はPlaywright/Vite上でWASMのみ）を疑い、`executionProviders: ['wasm']` を明示指定して切り分けを実施 → 症状変わらず、WebGPU/WASMの差異ではないと判明（この変更は原因ではなかったため後に取り消し、既定のプロバイダ選択に委ねる形へ戻した）
- [x] 0d.3 実際に生成されたクロップ画像をページ上に拡大表示して直接確認したところ、**カード下部の「所持〈数量〉」欄（紺色の帯）が一切クロップに含まれていない**ことが判明。検出領域の座標・サイズ自体はスパイクと一致していたが、クロップ内容が異なっていた
- [x] 0d.4 元画像のピクセルをサンプリングし、紺色の帯（R,G,B ≈ 99-119, 131-144, 162-169）が、当初の「カードらしい」判定条件（R>150 かつ G>150）を満たさず非カード領域として扱われ、行方向プロジェクションがカード上半分で打ち切られていたことを特定。**0b/0c のスパイク検証・単体テストはいずれもこの欠陥を検出できていなかった**（手動クロップ・自動検出のクロップ範囲双方でカード全体が偶然収まっていたため）
- [x] 0d.5 ページ背景・カード行間の隙間は「B-R ≥ 100」、カードの白背景・紺色の帯はいずれも「B-R ≤ 64」と明確に分離できることをピクセルサンプリングで確認し、`isCardish` の判定条件を「B-R ≤ 80」の単一閾値に変更（design.md Decision 4参照）
- [x] 0d.6 修正後、単体テスト（33件）が全てパスすることを確認
- [x] 0d.7 修正後、`pnpm dev` の実アプリで実スクリーンショット2枚を再アップロードし、フルE2Eフローを検証 → **18カード中16カードで品目名・所持数とも完全一致**（狂の魔石3052・剣の秘石81・弓の秘石506・槍の秘石328・騎の秘石668・術の秘石763・殺の秘石544・狂の秘石933・英雄の証79・凶骨86・竜の牙1669・虚影の塵132・愚者の鎖687・万死の毒針271・魔術髄液2207・宵哭きの鉄杭768）、残り2カード（励振火薬・赦免の小鐘）は正しく見切れ「要確認」として自動確定対象から除外された
- [x] 0d.8 「反映する」で確定操作を実行し、`localStorage['posession']` が更新されることを確認。ページを再読み込みしても値が保持され、`/quests` の「所持数が未入力の素材」件数が 92→77 に減少（新規入力した15件と一致。英雄の証は元々10という値が入力済みだったため「未入力」件数には含まれていなかった）、クエスト効率ランキング・効率ポイントが新しい所持数を反映して再計算されることを確認（5.4のブラウザ実機検証を兼ねる）
