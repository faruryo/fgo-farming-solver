## ADDED Requirements

### Requirement: 進捗報告画面の表示
データ同期（保存）またはソルバー実行が成功した際、ユーザーに進捗を報告する画面を表示しなければならない (SHALL)。

#### Scenario: ソルバー実行成功時の進捗表示
- **WHEN** ソルバー実行が成功し、結果ページに遷移したとき
- **THEN** 進捗データを取得し、進捗報告モーダルが表示される。

#### Scenario: 比較対象なし時の表示
- **WHEN** 過去スナップショットが存在しないか、指定期間のスナップショットが取得できないとき
- **THEN** 「登録お疲れ様」モードのメッセージとともにモーダルが表示される。

### Requirement: 期間別進捗比較
システムは、現在と過去（前回、1週間前、1ヶ月前）のデータを比較し、素材獲得数や目標達成率の推移を表示しなければならない (SHALL)。

#### Scenario: 期間切り替え
- **WHEN** 進捗報告画面で「1週間前」を選択したとき
- **THEN** 1週間前のデータと比較した進捗内容（獲得素材数、減少した不足数、サーヴァント成長等）が表示される。

### Requirement: total_ap 進捗の tier 判定
システムは、経過時間に対する自然回復AP（5分=1AP）を基準とし、`total_ap` の減少量を 4 段階に分類しなければならない (SHALL)。

#### Scenario: 大進捗の判定
- **WHEN** `deltaAp >= naturalRecoveryAp * 1.5` のとき
- **THEN** tier は `large` と判定される。

#### Scenario: 中進捗の判定
- **WHEN** `naturalRecoveryAp <= deltaAp < naturalRecoveryAp * 1.5` のとき
- **THEN** tier は `medium` と判定される。

#### Scenario: 小進捗の判定
- **WHEN** `0 < deltaAp < naturalRecoveryAp` のとき
- **THEN** tier は `small` と判定される。

#### Scenario: 進捗ゼロの判定
- **WHEN** `deltaAp <= 0` のとき
- **THEN** tier は `none` と判定される。

### Requirement: 新規入手サーヴァントによる必要AP増加分のオフセット
システムは、比較スナップショット間で新規に「所持」化されたサーヴァントを検出し、その rarity 別の推定APを `deltaAp` に足し戻して純粋進捗を計算しなければならない (SHALL)。

#### Scenario: ガチャ後の進捗評価
- **WHEN** 比較スナップショット間で chaldea state の `disabled` が `true` から `false` に変化したサーヴァントが存在するとき
- **THEN** そのサーヴァントの rarity に応じた事前計算済み推定APを `deltaAp` に加算する。
- **THEN** オフセット適用後の値で tier 判定を行う。

### Requirement: rarity 別の必要AP推定値の事前計算
システムは、各 rarity（1〜5）から代表サーヴァントを抽出して solver で必要APを計算し、その平均値（または中央値）を保持しなければならない (SHALL)。

#### Scenario: rarity-AP テーブルの生成
- **WHEN** 事前計算プロセスが実行されたとき
- **THEN** 各 rarity からランダムに複数体のサーヴァントを選択し、共通目標（再臨/スキル/アペンド）達成までの必要APを solver で算出する。
- **THEN** 結果の平均値または中央値を rarity-AP テーブルとして保持する。

#### Scenario: 高難易度アクセス考慮の2セット計算
- **WHEN** rarity-AP テーブルを生成するとき
- **THEN** 「高難易度なし」と「高難易度あり（オーディールコール、冠位戦含む）」の 2 セットを生成する。

### Requirement: 高難易度コンテンツアクセス可否の判定
システムは、ユーザーの選択クエストリスト（`checkedQuests`）から高難易度コンテンツ（オーディールコール、冠位戦）への到達可否を推測しなければならない (SHALL)。

#### Scenario: アクセス可否の推測
- **WHEN** 進捗計算を行うために rarity-AP テーブルを選択する必要があるとき
- **THEN** ユーザーの `checkedQuests` に該当コンテンツの代表クエストIDが含まれるかをチェックする。
- **THEN** 含まれる場合は「高難易度あり」テーブル、含まれない場合は「基本」テーブルを使用する。

### Requirement: サーヴァント成長度の差分表示
システムは、chaldea state のスキル＋アペンドレベル合計が前回スナップショットから増加した場合、その成長量を進捗として表示しなければならない (SHALL)。

#### Scenario: スキル/アペンド成長の表示
- **WHEN** 比較スナップショット間でいずれかのサーヴァントのスキル/アペンドレベル合計が増加したとき
- **THEN** 進捗報告画面に「○○ のスキルが ○ レベル上がりました」のような成長表示を行う。

### Requirement: 目標増加のポジティブ表示
システムは、目標 `total_ap` が増加した場合（新規育成対象追加など）、これを「より高い目標」としてポジティブに表示しなければならない (SHALL)。

#### Scenario: 目標増加時の表示
- **WHEN** 比較スナップショットの目標 `total_ap` が現在より小さいとき（=目標が上方修正された）
- **THEN** 「より高い目標を掲げました」というポジティブメッセージを表示する。

### Requirement: マシュによる進捗フィードバック
進捗報告画面において、マシュ・キリエライトの画像と進捗種別に応じたメッセージを表示しなければならない (SHALL)。

#### Scenario: tier 別マシュメッセージ
- **WHEN** 進捗 tier が `large` / `medium` / `small` / `none` のいずれかに判定されたとき
- **THEN** その tier に対応する事前定義済みマシュメッセージ群からランダムにメッセージが選択され、マシュの画像とともに表示される。

#### Scenario: 「お疲れ様」モード
- **WHEN** 過去スナップショットがない、または全指標で改善ゼロのとき
- **THEN** マシュの「登録お疲れ様」系メッセージが表示される。

### Requirement: 達成感の視覚的演出
進捗 tier に応じて、モーダル内の色・サイズ・装飾の強度を変化させなければならない (SHALL)。アニメーションは v1 では実装しない。

#### Scenario: tier 別ビジュアル
- **WHEN** tier が `large` のとき
- **THEN** 派手な配色と大きい数値表示で達成感を強調する。

#### Scenario: 控えめな表示
- **WHEN** tier が `small` または `none` のとき
- **THEN** 落ち着いた配色と通常サイズで控えめに表示する。
