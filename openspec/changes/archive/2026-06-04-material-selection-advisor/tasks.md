## 1. データ計算・最適化エンジンの実装 (Core Optimization Logic)

- [x] 1.1 `Drops` （ドロップデータベース）から各アイテムの「最良AP効率（AP/個）」および「最良周回数効率（周回数/個）」を逆算する軽量な計算ヘルパーを実装する。
- [x] 1.2 貪欲法（Greedy Algorithm）を用いて、入力された候補素材、不足数、獲得可能総数から最適な配分数を計算する最適化ロジック（APモード/周回数モードに対応）を実装する。
- [x] 1.3 ドロップデータが存在しないアイテム（QPや特殊素材）が選択された場合の例外処理（節約期待値0、警告表示など）を追加する。
- [x] 1.4 計算ロジックを検証するための単体テストを `lib/material-selection-advisor.test.ts` に作成し、動作を検証する。
