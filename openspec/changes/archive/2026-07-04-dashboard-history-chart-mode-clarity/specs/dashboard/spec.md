## ADDED Requirements

### Requirement: 計算履歴グラフの表示モード明示と切り替え
計算履歴のグラフ（HistoryGraph）は、履歴に「通常」と「ストック込み」の両方の計算が存在する場合、無言でどちらか一方のデータ点を除外してはならない (SHALL)。その場合は現在表示中の種別が視認できる切り替えトグルを表示し、ユーザーがページ遷移せずにもう一方の種別へ切り替えられなければならない (SHALL)。

#### Scenario: 両種別が混在する場合はトグルを表示
- **WHEN** ログインユーザーの計算履歴に「通常」と「ストック込み」の両方の計算結果が存在するとき
- **THEN** ダッシュボードの計算履歴グラフに「通常 / ストック込み」の切り替えトグルが表示され、初期表示は最新の計算の種別に一致する。

#### Scenario: トグル操作でグラフとデータが切り替わる
- **WHEN** ユーザーがダッシュボードの計算履歴グラフ上でトグルを操作したとき
- **THEN** グラフは選択された種別の計算履歴のみで再描画され、予測線もその種別のデータのみから再計算される。
- **THEN** トグルの活性表示（ハイライト）は現在選択中の種別と一致する。

#### Scenario: 片方の種別のみの場合はトグルを表示しない
- **WHEN** ログインユーザーの計算履歴が「通常」または「ストック込み」いずれか一方のみで構成されているとき
- **THEN** トグルは表示されず、存在する種別の履歴のみでグラフが描画される。
