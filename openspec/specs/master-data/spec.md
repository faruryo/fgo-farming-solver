# 仕様書: マスターデータ管理 (Master Data Management)

## 概要
FGO周回ソルバーが必要とする最新のアイテム情報、クエスト情報、およびドロップ率データを外部ソースから取得し、アプリケーションが利用可能な形式に変換・保持する機能です。

## 要件

### 1. データソース
- **ドロップ率データ**: 「FGOアイテム効率劇場～2～」スプレッドシート（CSV形式）から取得する。
- **アイテムメタデータ**: Atlas Academy API (`nice_item.json`) から正式名称やアイテムIDを取得する。

### 2. アイテム名のマッピング
スプレッドシート上の日本語略称を、Atlas Academy の正式なアイテム名に変換する。
- **変換ルール**: 
  - 静的なマッピングテーブル（例: `証` -> `英雄の証`）
  - クラス・種別パターン（例: `[クラス名][輝/魔/秘/ピ/モ]` の自動変換）
  - 部分一致によるフォールバック

### 3. データ最適化 (Top 5 フィルタリング)
ソルバーの計算速度を維持し、データサイズを Cloudflare KV の制限内に収めるため、以下の処理を行う。
- アイテムごとに、ドロップ率が高い上位5つのクエストのみを保持する。
- 上位5位に含まれないクエストデータは最終的なデータセットから除外する。

### 4. 更新タイミング
- **自動更新**: GitHub Actions の Cron ジョブにより1日1回実行。
- **手動更新**: 管理用エンドポイント `/update` へのリクエストにより即時実行。

## データ構造 (JSON)
`MASTER_DATA` KV 内の `all_drops_json` キーに、以下の構造で保存される。

- `items`: アイテムID、名称、カテゴリ
- `quests`: クエストID、名称、エリア、AP消費量
- `drop_rates`: クエストID、アイテムID、ドロップ率

## 制約
- AP 5 未満のクエストは計算上のノイズを避けるため除外する。
- Atlas Academy API で定義されていないアイテムは、マッピング対象外としてスキップされる。
